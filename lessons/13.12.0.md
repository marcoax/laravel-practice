# Lesson 13.12 — Scheduler metadata & opt-out listener discovery

> Version: 13.12.0 (May 26, 2026) · Type: new · Source: laravel-news.com/laravel-13-12-0

## What changed
Two complementary control points:

- **Scheduler `withAttributes()`** — attach arbitrary key/value metadata to a scheduled
  event, readable in lifecycle callbacks instead of parsing the command string.
  ```php
  $schedule->command('audio:import-podcasts')
      ->withAttributes(['tag' => 'import-premium-podcasts']);
  ```
- **`ShouldBeDiscovered`** — an interface that lets an auto-discovered event listener
  decide, at boot, whether it registers at all.
  ```php
  public static function shouldBeDiscovered(): bool {
      return app()->environment('production');
  }
  ```

Also in 13.12: `Worker::$stopOnLostConnection = false` (no restart loop when a DB
connection drops), SQLite `file:` URI connections, and `Str::studly()`/`Str::pascal()`
normalisation fixes.

## Why it might matter
`withAttributes()` makes scheduled tasks **observable** — you can tag tasks for metrics or
log filtering without string-matching command names. `ShouldBeDiscovered` stops listeners
(and their queued jobs) from registering in environments where they shouldn't fire, which
is cleaner than guarding inside `handle()` after the job is already dispatched.

## Try it
- [ ] In eraCms, tag a couple of `Kernel`/`schedule()` entries with `withAttributes()` and
      read them in a `->before()`/`->after()` hook for logging.
- [ ] Pick a listener that should only run in production and convert an in-`handle()`
      environment guard into `ShouldBeDiscovered::shouldBeDiscovered()`.

## Relevant to your project?
- How many scheduled commands does eraCms run, and do you currently grep logs by command
  name? (Tags replace that.)
- Are there listeners doing `if (! app()->isProduction()) return;` that could opt out of
  discovery entirely?

_Example — eraCms:_ the listener opt-out is the higher-value half — with ~79 models there
are likely event listeners you'd rather not register outside production at all.
