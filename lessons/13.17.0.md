# Lesson 13.17 — First-class route metadata

> Version: 13.17.0 (Jun 24, 2026) · Type: new · Source: laravel-news.com/laravel-13-17-0

## What changed
Routes get first-class `metadata()`: structured data attached to a route that survives the
whole pipeline — including **route caching** and **group inheritance**. Previously you'd
stash this informally inside the route action array.

```php
Route::get('/users', [UserController::class, 'index'])
    ->metadata(['head' => ['title' => 'Users']]);

// later, in middleware / a view composer:
$request->route()->getMetadata('head.title'); // 'Users'
```

Metadata set on a `Route::group(...)` cascades to child routes, with recursive merging for
associative arrays — so shared defaults live on the group and children override leaf keys.

## Why it might matter
Route-level config (SEO `<head>` data, access tiers, doc tags) finally has a typed,
cache-safe home that inherits down groups instead of being duplicated per route or smuggled
through action arrays. Read it once in middleware or a composer rather than mapping route
names to a side table.

## Try it
- [ ] In eraCms, pick a group of routes that share a layout/section and put common
      metadata (section tag, page title prefix) on the `Route::group`, overriding per leaf.
- [ ] Read `getMetadata()` in a middleware or view composer to drive `<head>` / breadcrumbs
      instead of a route-name → config lookup table.

## Relevant to your project?
- Does eraCms currently map route names to metadata in a config array or DB table? Inline
  route metadata could replace that.
- Are your routes cached in production? (Metadata persists through caching — that's the
  point versus stuffing closures in the action array.)

_Example — eraCms:_ promising for the Vue-rendered admin if per-section `<head>`/breadcrumb
data is today derived from route names; the group cascade fits a sectioned admin well.
