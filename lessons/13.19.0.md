# Lesson 13.19 ‚Äî HTTP QUERY method support

> Version: 13.19.0 (Jul 8, 2026) ¬∑ Type: new ¬∑ Source: laravel-news.com/laravel-13-19-0

## What changed
The framework rounds out support for the **HTTP QUERY verb** ‚Äî a safe, idempotent method
that carries its parameters in the request *body* instead of the URL. The HTTP client gains
`Http::query()`, mirroring `post()`/`put()`/`patch()` (JSON by default, form-encoded via
`asForm()`), and the test suite gains matching `query()` / `queryJson()` helpers:

```php
// Client
$response = Http::query('https://api.example.com/search', [
    'filter' => ['status' => 'active'],
]);

// Test
Route::match(['QUERY'], '/search', fn () => request()->input('filter'));

$this->queryJson('/search', ['filter' => 'active'])->assertOk();
```

Alongside it, a batch of smaller additions:

- **`reduceInto()`** ‚Äî a `reduce()` variant that mutates an accumulator in place; the
  callback's return value is ignored, so object accumulators need no `return`
  (array/scalar accumulators are taken by reference):

  ```php
  $stats = $orders->reduceInto(new OrderStats, function ($stats, $order) {
      $stats->total += $order->amount;
      $stats->count++;
  });
  ```

- **`Str::counted()`** (and its `Stringable` twin) ‚Äî pluralize and prepend the count in
  one call: `str('order')->counted(2); // "2 orders"`.
- **Queue** ‚Äî bulk SQS dispatches now use the `SendMessageBatch` API (one call instead of
  one per job), and the queue fake can inspect *reserved* jobs (picked up, not yet done).
- Fixes: `deletedAtColumn` passes through to `assertSoftDeleted()` /
  `assertNotSoftDeleted()`, mail config options without a name, semicolon-terminated
  default styles in `ComponentAttributeBag::merge()`.

## Why it might matter
Search endpoints with rich filters hit the limits of query strings quickly ‚Äî URL length
caps, encoding pain for nested arrays, filters leaking into access logs. QUERY gives those
requests the semantics of a GET (safe, cacheable in principle) with the ergonomics of a
POST body, and Laravel now supports it end to end: route matching, client, and test
helpers. `reduceInto()` cleans up the common "reduce with an object accumulator" pattern
where every iteration had to `return $acc;` for no reason.

## Relevant to your project?
- Does {{reference_project}} have API search endpoints with complex or nested filters that today go
  through POST-as-search or long query strings? QUERY is the semantically honest verb for
  them ‚Äî but check that your HTTP infrastructure (proxies, CDN, clients) passes the verb
  through.
- Do you dispatch jobs in bulk on SQS? The `SendMessageBatch` change is free throughput ‚Äî
  no code change, fewer API calls.
- Do your queue tests need to assert on in-flight jobs? The reserved-job inspection on the
  fake covers a spot that previously required real workers.
