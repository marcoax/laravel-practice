# Lesson 13.15 — Typed translation accessors

> Version: 13.15.0 (Jun 9, 2026) · Type: new · Source: laravel-news.com/laravel-13-15-0

## What changed
`trans()` gains concrete typed accessors — `trans()->string()` and `trans()->array()` —
that return a definite `string` / `array` instead of the old broad union type. It mirrors
the typed helpers already on `config()` and `request()`.

```php
public function label(): string {
    return trans()->string($this->name);
}

public function options(): array {
    return trans()->array($this->options_key);
}
```

## Why it might matter
The classic `__()`/`trans()` return type is loose, so static analysers (PHPStan/Larastatic,
Psalm) flag every place a translation flows into a strictly-typed method. The typed
accessors remove the `// @var string` casts and `(string)` coercions you'd otherwise
sprinkle around to keep the analysis clean — the type is asserted at the source.

## Try it
- [ ] In eraCms, find a spot where a translation is cast or annotated to satisfy a type hint
      and replace it with `trans()->string(...)`.
- [ ] If you load a group of strings (e.g. a select's options) from a lang file, switch the
      lookup to `trans()->array(...)` and drop any manual array assertion.

## Relevant to your project?
- Does eraCms run PHPStan/Psalm at a level where loose `trans()` returns actually surface?
- How many translation lookups feed directly into typed method signatures or DTO
  constructors today?

_Example — eraCms:_ worth it where translations feed typed signatures (DTOs, accessor
methods) and PHPStan flags the loose returns; otherwise cosmetic — adopt it opportunistically
as you touch i18n code, not as a sweep.
