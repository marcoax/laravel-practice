# Lesson 13.9 — HTML password rules from your validation rules

> Version: 13.9.0 (May 13, 2026) · Type: new · Source: laravel-news.com/laravel-13-9-0

## What changed
Laravel 13.9 adds `Password::toPasswordRulesString()`, which renders a `Password`
validation rule object into the browser's `passwordrules` attribute string. Define your
policy once and emit it into the markup so password managers (and Safari/iOS) generate a
compliant password on the first try instead of failing your server-side validation.

```php
Password::defaults(fn () => Password::min(12)->max(64)->mixedCase()->numbers()->symbols());

// In a Blade view:
<input type="password" passwordrules="{{ Password::defaults()->toPasswordRulesString() }}">
// → minlength: 12; maxlength: 64; required: lower; required: upper; required: digit; required: special;
```

The same release also ships `PendingDispatch::when()`/`unless()`, a `PreparesForDispatch`
interface, `foreignUuidFor()`, and optional disk storage for oversized SQS payloads — but
the password-rules helper is the headline.

## Why it might matter
It removes a class of friction: the user types a password, the manager suggests one that
matches, and your `Password` rule and the browser hint never drift apart because they come
from the **same** object. One source of truth for the policy, on both ends of the form.

## Try it
- [ ] In eraCms, find the registration / password-reset form and add a `passwordrules`
      attribute fed by your existing `Password` rule.
- [ ] Change the policy in one place (e.g. bump `min(12)`) and confirm both the server
      rule and the emitted attribute move together.

## Relevant to your project?
- Does eraCms centralise its password policy via `Password::defaults()`, or is it inline
  per-form? (The helper pays off most when there's a single defaults closure.)
- Do your admin users rely on password managers where the hint actually helps?

_Example — eraCms:_ worth it for the public-facing auth forms if you already declare a
`Password::defaults()` closure; low value on internal-only screens.
