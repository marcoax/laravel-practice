# Lesson 13.13 — Bulk job dispatch with Bus::bulk()

> Version: 13.13.0 (Jun 2, 2026) · Type: new · Source: laravel-news.com/laravel-13-13-0

## What changed
`Bus::bulk()` dispatches many jobs in a single batch operation rather than looping and
queueing them one at a time, cutting the per-job round-trips to the queue backend.

```php
Bus::bulk([
    new ProcessImage($a),
    new ProcessImage($b),
    new ProcessImage($c),
]);
```

The release also adds storage-backed mail attachments — `attachFromStorage()` and
`attachFromStorageDisk()` on `MailMessage` — and lets the HTTP client act as a PSR-18
client.

## Why it might matter
When you enqueue hundreds of jobs in a request (imports, fan-out notifications),
dispatching them individually is a chatty hot path against Redis/SQS. `Bus::bulk()`
collapses that into one operation — fewer network round-trips, less latency on the
dispatching request.

## Try it
- [ ] Find a loop in eraCms that does `SomeJob::dispatch(...)` per item and convert it to a
      single `Bus::bulk([...])`.
- [ ] If you send mail with attachments pulled from a disk, swap the manual
      `Storage::get()` + `->attachData()` for `attachFromStorageDisk()`.

## Relevant to your project?
- Where does eraCms enqueue jobs in a loop — bulk imports, per-model reindexing,
  notification fan-out? Those are the `Bus::bulk()` candidates.
- Do any mailables attach files you currently read off a disk by hand?

_Example — eraCms:_ likely useful around model imports / search reindexing across ~79
models, where one request can spawn a long list of jobs.
