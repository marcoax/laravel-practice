Learner-selected discovery of authoritative deeper resources

2026-07-03 · outcome of a /grill-with-docs session · source issue: marcoax/laravel-practice#29 · ADR-0019

1. Goal

At the end of a lesson, the learner can explicitly ask the agent to search the web and YouTube for very authoritative deeper resources on that lesson's topic. Accepted resources are added both to the current lesson HTML and to RESOURCES.md. The feature is opt-in per lesson and must not turn every lesson into a noisy bibliography.

2. Decisions (grilled and settled)

D1 — The learner chooses; the agent does not auto-trigger on "high-value" lessons. The original issue framed this as an offer for lessons the agent flags as high-value. We reject that trigger. The offer appears at the end of every lesson, and the learner decides whether to run discovery. The agent may still explain when the web looks thin, but it does not hide the choice behind its own scoring.
D2 — End-of-lesson placement. The prompt appears after the normal teaching loop for the lesson: explanation, practice, recall/quiz, and the ordinary "go deeper on a question" invitation. If accepted, resource discovery runs before the final lifecycle writes so the learning record can mention it if useful. Failure or an empty high-authority result set never blocks lesson completion.
D3 — Very authoritative sources only. Prefer official docs, official package repositories, release notes, framework source, and clearly recognized Laravel/PHP authors or conference talks. Reject generic SEO tutorials, listicles, undated API content, shallow YouTube videos, and marketing dressed as education. Every accepted link must say why it is authoritative and why it is worth the learner's time.
D4 — Two destinations. Add the curated links into the current lesson HTML for immediate use, and into RESOURCES.md as the durable workspace index. If no source clears the bar, make no file changes and say so in chat.
D5 — Hard cap: four resources per lesson. Maximum 4 resources: up to 2 official/primary sources and up to 2 community/talk/video sources. At least one official or primary source is preferred when one exists. Bare links are never accepted.

3. User-facing flow

Use the learner's configured chat language from learning-config.md. For the current learner that means Italian. The prompt should be short and predictable:

Vuoi che cerchi risorse web/YouTube molto autorevoli su questo tema
e le aggiunga alla lezione + RESOURCES.md?
  • No / ignored: no search and no resource-file changes; continue the lesson lifecycle gate.
  • Yes: search web and YouTube, inspect candidates, keep only sources that clear the authority bar, then patch files.
  • Thin web: if the topic is too new or no source clears the bar, say that clearly in chat and leave files untouched.

4. Resource quality contract

AcceptReject
Laravel official docs, package docs, GitHub repositories, release notes, framework source. Generic SEO tutorials, copied examples, listicles, shallow "what is X" posts.
Recognized Laravel/PHP authors, conference talks, maintainers, first-party ecosystem channels. Unknown YouTube channels with no demonstrated authority or videos without concrete technical content.
Fresh material for APIs that are still moving; older material only when it is stable and still accurate. Undated content for fast-moving APIs, outdated package names, beta-era behavior presented as current.

For each accepted resource record:

  • title;
  • URL;
  • type: official docs, source, article, talk, video;
  • author/channel/publisher;
  • why it is authoritative;
  • why it helps this specific lesson.

5. File contract

Lesson HTML

Add a compact section near the existing Approfondisci / primary-source area. Reuse assets/lesson.css; add a small reusable component only if current .primary-source and .deepen styles cannot express the block cleanly.

<section class="curated-resources">
  <h2>Risorse autorevoli per approfondire</h2>
  <ul>
    <li>
      <a href="...">Title</a>
      <span class="resource-meta">Official docs · Laravel</span>
      <p>Why authoritative; why it matters for this lesson.</p>
    </li>
  </ul>
</section>

RESOURCES.md

Create it lazily on first accepted resource discovery. It is per-user `/teach` workspace state, like rendered lessons and learning records, so implementation should add /RESOURCES.md to .gitignore. Keep the file in English, matching project documentation rules.

6. Implementation steps

  1. edit CLAUDE.md — lesson lifecycle: after recall/deep-dive and before final lifecycle writes, offer learner-selected resource discovery. State that decline or empty results are non-blocking.
  2. edit NOTES.md — teaching preference: offer resource discovery at the end of each lesson; only very authoritative sources; maximum 4 resources.
  3. edit .gitignore — add /RESOURCES.md as per-user `/teach` workspace state.
  4. edit lessons/_template.md — document the optional curated-resources HTML block separately from the single primary-source callout.
  5. edit assets/lesson.css — add .curated-resources styles only if the existing callout styles are insufficient; keep it printable and compact.
  6. edit CONTEXT.md — already started in this design session: keep the terms learner-selected resource discovery and RESOURCES.md aligned with the final implementation.
  7. new docs/adr/0019-learner-selected-resource-discovery.md — record the trigger and curation decision.
  8. manual Retrofit lesson 10 (AI SDK) as the first example only after the flow is documented: ask the learner at lesson end, then search and patch if accepted.

7. Out of scope

  • Automatic "high-value lesson" scoring by the agent.
  • Adding links when the learner declines.
  • Generic resource harvesting or broad bibliographies.
  • Creating a search service, crawler, cache, or automated ranking system.
  • Making RESOURCES.md the source of truth for generated lesson content retroactively.
  • Editable resource management UI in index.html.

8. Manual verification

  • Finish a lesson and confirm the resource-discovery prompt appears at the end in the learner's chat language.
  • Decline the prompt and verify no lesson HTML or RESOURCES.md change is made by this feature.
  • Accept on a topic with strong official docs; verify the lesson HTML gains at most 4 annotated links and RESOURCES.md gains the matching entries.
  • Accept on a topic with weak or noisy search results; verify the agent reports no sufficiently authoritative sources and leaves files untouched.
  • Check every accepted YouTube/video link includes channel/author and a reason it clears the authority bar.
  • Confirm RESOURCES.md is ignored by git and does not leak per-user learning state into the tracked template.
